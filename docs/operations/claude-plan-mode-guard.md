# Claude Plan Mode Guard for Polecats

## Operational rule

Gas Town **polecats** are autonomous, implementation-first workers. They must
never run Claude Code in `--permission-mode plan`, because plan mode forces the
agent to stop and ask for approval before using tools. A polecat that stops for
approval cannot make progress on its hook and stalls the whole rig.

The repo enforces this rule in three ways:

1. **Launch-time normalization** — every Claude-compatible polecat runtime has
   its command-line args normalized to include
   `--permission-mode bypassPermissions`. If an alias, `role_agents` override,
   or explicit `--agent` flag passes `--permission-mode plan`, the plan flag is
   removed and replaced with `bypassPermissions`.

2. **Runtime detection** — the witness patrol inspects live polecat processes
   for `--permission-mode plan`. If a polecat is observed in plan mode (for
   example, because a process was started outside of Gas Town or a config file
   was hand-edited), it is classified as a recoverable
   `plan-mode` zombie and restarted. The patrol receipt records the exact
   permission-mode value so operators have clear evidence.

3. **Hooks-level default** — Claude Code settings generated by Gas Town set
   `permissions.defaultMode` to `bypassPermissions` for all managed
   configurations, providing defense-in-depth against interactive-mode regressions.

## Scope

- **Applies to**: `role=polecat` with a Claude-compatible runtime. A runtime is
  Claude-compatible when its resolved command is `claude` (or a path ending in
  `/claude`) or when its provider is explicitly `claude`.
- **Does not apply to**: non-polecat roles such as `mayor`, `witness`,
  `refinery`, `crew`, or non-Claude runtimes (Codex, Gemini, OpenCode, etc.).
  Planner/reviewer-style Claude aliases used for non-polecat roles may keep
  `--permission-mode plan` if that is intentional.

## Configuration examples

### Default (no config)

A polecat launched with the default Claude preset receives:

```
claude --dangerously-skip-permissions --settings <path> --permission-mode bypassPermissions
```

### Custom Claude alias with explicit plan mode

A town or rig setting like:

```json
{
  "agents": {
    "claude-planner": {
      "command": "claude",
      "args": ["--permission-mode", "plan", "--model", "sonnet"]
    }
  },
  "role_agents": {
    "polecat": "claude-planner"
  }
}
```

is normalized at launch to:

```
claude --model sonnet --settings <path> --permission-mode bypassPermissions
```

The same alias assigned to a non-polecat role keeps `--permission-mode plan`.

## Health evidence

When the witness detects a plan-mode polecat, the patrol receipt contains:

```json
{
  "rig": "gastown",
  "polecat": "jasper",
  "verdict": "stale",
  "recommended_action": "restarted-plan-mode-polecat (permission-mode=plan)",
  "evidence": {
    "classification": "plan-mode",
    "hook_bead": "gastown-fbg"
  }
}
```

This provides the Mayor and Witness with an auditable signal: the polecat was
recoverable, the cause was a plan-mode launch, and the remediation was a restart
without losing the worktree.
